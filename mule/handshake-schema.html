Note on creation of resources and references
Some creation methods allow to specify nested objects either as new or references to existing.
Whenever you want to create a object that references an existing one (for example, a new Order for an existing Customer), you just have to specify the resourceUri of the nested object.
&lt;pre&gt;
&amp;lt;handshake:create-order&amp;gt;
&amp;lt;handshake:order&amp;gt;
&amp;lt;handshake:customer resourceUri="/api/v2/customers/1"/&amp;gt;
&amp;lt;/handshake:create-order&amp;gt;
&amp;lt;/handshake:order&amp;gt;
&lt;/pre&gt;
However, you can also create a nested object alongside the master one.
&lt;pre&gt;
&amp;lt;handshake:create-order&amp;gt;
&amp;lt;handshake:order&amp;gt;
&amp;lt;handshake:customer id="SOME_ID" name="Some Customer"&amp;gt;
&amp;lt;handshake:bill-to city="Springfiled" stree="742 Evergreen Terrace"/&amp;gt;
&amp;lt;/handshake:customer&amp;gt;
&amp;lt;/handshake:create-order&amp;gt;
&amp;lt;/handshake:order&amp;gt;
&lt;/pre&gt;
When you create a new object through the API, the returned value will always contain a resourceUri attribute, which should be stored in your external system for future use as a reference.</xs:documentation>
